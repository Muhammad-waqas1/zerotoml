This project follows a simple rule: **be kind and constructive**. All participants are expected to treat each other with respect. Mathematical debates are welcome; personal attacks are not. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for the full policy.

## Getting Started

### Prerequisites

- Git
- Python 3.8+ (only for regenerating lessons)
- A browser (to preview changes)

### Fork and Clone

```bash
# 1. Fork the repo on GitHub (click the Fork button)

# 2. Clone your fork
git clone https://github.com/YOUR_USERNAME/zerotoml.git
cd zerotoml

# 3. Add the upstream remote
git remote add upstream https://github.com/Muhammad-waqas1/zerotoml.git

# 4. Create a branch for your change
git checkout -b fix/lesson-04-svm-kernel-explanation
# or
git checkout -b feat/lesson-10-transformers
```

### Preview the Site

No build step required:

```bash
# Python simple server
python3 -m http.server 8080
# Visit http://localhost:8080

# OR just open index.html directly in Chrome/Firefox
```

## How Lessons Are Built

All lesson HTML is **generated by Python scripts**, not hand-written. This keeps every page consistent — same layout, same sidebar, same ToC, same prev/next navigation — without copy-pasting hundreds of lines.

```
generate_lessons_v2.py  →  lessons/01-*.html  and  lessons/02-*.html  and  lessons/03-*.html
generate_lessons_v3.py  →  lessons/04-*.html  and  lessons/05-*.html
generate_lessons_v4.py  →  lessons/06-*.html  and  lessons/07-*.html
generate_lessons_v5.py  →  lessons/08-*.html  and  lessons/09-*.html
```

Each script contains:

1. **`ALL_MODULES`** — the sidebar curriculum (all 9 modules, used on every lesson page)
2. **Content strings** — e.g. `C04_REGRESSION`, `C07_PERCEPTRON` — raw HTML for each lesson body
3. **`PAGES` list** — metadata for each lesson (filename, title, description, module, difficulty, read time, prev/next links)
4. **`wrap()` function** — assembles the full HTML page from layout + content

**You only ever edit content strings and the PAGES list.** The layout, sidebar, ToC, and navigation are generated automatically.

## Editing an Existing Lesson

### Step 1 — Find the content string

Open the appropriate generator script and search for the lesson's content variable.

```bash
# Example: editing the SVM lesson (Lesson 4.3)
# → Open generate_lessons_v3.py
# → Find  C04_SVM = """..."""
grep -n "C04_SVM" generate_lessons_v3.py
```

### Step 2 — Edit the HTML string

Content strings are raw HTML wrapped in triple quotes. Edit them directly. The HTML uses custom CSS classes from `assets/css/main.css`.

```python
# Example: fixing a sentence in C04_SVM
C04_SVM = """
          <h2 id="intuition">The Margin Idea</h2>
          <p>Your corrected explanation here...</p>
          ...
"""
```

> ⚠️ **Avoid triple quotes inside content strings.** Python will end the string early. Use `&#34;` for `"` inside HTML attributes if needed, or single quotes.

### Step 3 — Regenerate the HTML

```bash
python3 generate_lessons_v3.py  # for modules 04-05
# (or v2, v4, v5 for other modules)
```

### Step 4 — Preview

Open `lessons/04-svm.html` in a browser and verify your changes look correct.

## Writing a New Lesson

### Step 1 — Write the content string

Add a new content variable to the appropriate generator script (or create `generate_lessons_v6.py` for new modules):

```python
C05_NEW_LESSON = """
          <h2 id="introduction">Introduction</h2>
          <p>Your opening paragraph — intuition first, always.</p>

          <h2 id="the-math">The Mathematics</h2>
          <div class="math-block">
            <span class="math-label">Key Equation</span>
            $$y = f(x; \\theta)$$
            <span class="math-caption">Explain every symbol here.</span>
          </div>

          <h2 id="implementation">Python Implementation</h2>
          <div class="code-block">
            <div class="code-header">
              <span class="code-lang">Python</span>
              <button class="copy-btn" onclick="copyCode(this)">Copy</button>
            </div>
<pre><code class="language-python">import numpy as np
# Your code here
</code></pre>
          </div>

          <h2 id="summary">Summary</h2>
          <ul class="lesson-list">
            <li>Key takeaway 1.</li>
            <li>Key takeaway 2.</li>
          </ul>"""
```

### Step 2 — Add to the PAGES list

```python
PAGES = [
    # ...existing pages...
    (
        "05-new-lesson.html",              # filename
        "Your Lesson Title",               # <title> and <h1>
        "One sentence that describes this lesson for SEO meta description.",
        "Module 05",                       # badge label
        "Intermediate",                    # "Beginner" or "Intermediate"
        "18 min",                          # estimated read time
        "Module 05 — Unsupervised Learning",  # breadcrumb
        C05_NEW_LESSON,                    # content string
        "05-association.html",             # prev lesson filename (or None)
        "5.3 Association Rule Learning",   # prev lesson label (or None)
        None,                              # next lesson filename
        None,                              # next lesson label
    ),
]
```

### Step 3 — Update the sidebar

Add the new lesson to `ALL_MODULES` in the generator script so it appears in the sidebar on every page:

```python
("mod-5", "05", "Unsupervised Learning", [
    ("05-clustering.html",     "5.1 Clustering"),
    ("05-dimreduction.html",   "5.2 Dimensionality Reduction"),
    ("05-association.html",    "5.3 Association Rule Learning"),
    ("05-new-lesson.html",     "5.4 Your New Lesson"),    # ← add this
]),
```

### Step 4 — Update ALL generator scripts

The sidebar is rendered identically across all lesson pages, including those generated by other scripts. Add the new lesson to `ALL_MODULES` in **every** generator script (`v2.py`, `v3.py`, `v4.py`, `v5.py`, `v6.py`) and regenerate all of them.

### Step 5 — Regenerate and preview

```bash
python3 generate_lessons_v3.py
python3 generate_lessons_v2.py  # also update other scripts' sidebars
python3 generate_lessons_v4.py
python3 generate_lessons_v5.py
```

### Lesson Quality Checklist

Before submitting:

- [ ] Starts with intuition/motivation, not definitions
- [ ] Every mathematical symbol is defined before use
- [ ] Every equation is in a `math-block` div with a label and caption
- [ ] Code is complete and runnable (not just fragments)
- [ ] Code has meaningful variable names and inline comments
- [ ] Summary section lists the key takeaways
- [ ] `id` attributes on all `<h2>` and `<h3>` tags (for ToC auto-generation)
- [ ] No triple quotes inside the content string
- [ ] Tested in Chrome and Firefox

## HTML Component Reference

These CSS classes are available in all lesson content strings.

### Section heading with anchor
```html
<h2 id="your-section">Your Section Title</h2>
<h3 id="your-subsection">Your Subsection</h3>
```

### Math equation block
```html
<div class="math-block">
  <span class="math-label">Equation Name</span>
  $$\hat{y} = \mathbf{w}^\top \mathbf{x} + b$$
  <span class="math-caption">Explain the equation here.</span>
</div>
```
Use `$$...$$` for display (block) math, `$...$` for inline math. Double-escape backslashes: `\\sigma` not `\sigma`.

### Code block
```html
<div class="code-block">
  <div class="code-header">
    <span class="code-lang">Python</span>
    <button class="copy-btn" onclick="copyCode(this)">Copy</button>
  </div>
<pre><code class="language-python">your code here
</code></pre>
</div>
```
Also supported: `language-bash`, `language-yaml`, `language-docker`.

### Callout boxes
```html
<!-- Info (blue) -->
<div class="callout callout--info">
  <div class="callout-icon">💡</div>
  <div><strong>Note title:</strong> Your note text.</div>
</div>

<!-- Warning (terracotta) -->
<div class="callout callout--warning">
  <div class="callout-icon">⚠️</div>
  <div><strong>Warning:</strong> Your warning text.</div>
</div>

<!-- Tip (sage green) -->
<div class="callout callout--tip">
  <div class="callout-icon">🌿</div>
  <div><strong>Tip:</strong> Your tip text.</div>
</div>
```

### Definition list (term → description)
```html
<div class="def-list">
  <div class="def-row">
    <span class="def-term">Term</span>
    <span class="def-desc">Description of this term.</span>
  </div>
</div>
```

### Comparison grid (two-column)
```html
<div class="comparison-grid">
  <div class="comparison-col">
    <h4 class="text-xs font-mono uppercase tracking-wider mb-2" style="color:var(--text-muted)">Option A</h4>
    <p>Description of option A...</p>
  </div>
  <div class="comparison-col">
    <h4 class="text-xs font-mono uppercase tracking-wider mb-2" style="color:var(--text-muted)">Option B</h4>
    <p>Description of option B...</p>
  </div>
</div>
```

### Bullet list (styled)
```html
<ul class="lesson-list">
  <li>First point.</li>
  <li>Second point with <strong>bold</strong> text.</li>
</ul>
```

### Numbered list (styled)
```html
<ol class="lesson-list">
  <li>Step one.</li>
  <li>Step two.</li>
</ol>
```

## Code Style

### Python examples in lessons

- Use `numpy as np`, `pandas as pd`, `matplotlib.pyplot as plt` — always these aliases
- Use `np.random.default_rng(42)` for reproducibility (preferred over `np.random.seed`)
- Include `print()` statements so readers see expected output
- Prefer complete, runnable examples over fragments
- Keep examples focused — one concept per block
- Add inline comments for non-obvious lines: `# shape (m, n) — m samples, n features`
- Use f-strings: `print(f"Accuracy: {acc:.4f}")`

### HTML in content strings

- Indent content with 10 spaces (matches the `<article>` indentation in `wrap()`)
- Always add `id` attributes to `<h2>` and `<h3>` — the JS uses them for the ToC
- IDs should be lowercase, hyphenated: `id="gradient-descent"` not `id="GradientDescent"`
- Don't use `<br>` for spacing — use paragraph tags or margin classes
- Don't inline styles unless necessary — use existing CSS classes

### Commit messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
fix: correct MSE formula in lesson 4.1
feat: add lesson 5.4 - Gaussian Mixture Models
docs: improve intro paragraph in lesson 7.3
style: fix indentation in lesson 6.2 code block
refactor: update sidebar in all generator scripts
```

## Pull Request Process

1. **Check existing issues** before starting — someone may already be working on it
2. **Keep PRs focused** — one lesson or one fix per PR makes review easier
3. **Fill out the PR template** — describe what changed and why
4. **Regenerate affected lesson pages** before submitting
5. **Test in a browser** — verify math renders, code blocks look correct, sidebar is intact

### What happens after you submit

- A maintainer will review the PR within a few days
- Minor style suggestions may be requested — this is normal
- Once approved, it will be merged to `main` and deployed automatically to GitHub Pages

## Reporting Bugs

Use [GitHub Issues](https://github.com/Muhammad-waqas1/zerotoml/issues) with the appropriate template:

- **Bug report** — broken layout, wrong math, code that doesn't run, broken link
- **Content improvement** — unclear explanation, better example, missing context
- **Feature request** — new lesson idea, UI improvement, new functionality

Please include:
- Which lesson/page is affected (URL or filename)
- What you expected to see
- What you actually see
- Browser and OS (for layout bugs)
